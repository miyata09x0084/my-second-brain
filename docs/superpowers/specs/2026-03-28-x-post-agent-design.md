# X投稿エージェント設計書

日付: 2026-03-28

## 概要

Xのアルゴリズムに適合する価値のある投稿を生成するマルチエージェントスキル。Web検索でトレンドを収集し、批判的分析を経て、3つの投稿案を生成する。

## 発信テーマ

- テック・エンジニアリング（技術知見、開発Tips、アーキテクチャ考察）
- AI・プロダクティビティ（AI活用術、生産性向上の実践）

## トーン・キャラクター

優先度: 問いかけ > 実務共有 > 解説

会話を生むことを軸に、実体験ベースの共有と噛み砕いた解説を織り交ぜる。

## アーキテクチャ

```
ユーザー入力（テーマ or インプット or なし）
        │
   Phase 1: ネタ探索（マルチエージェント）
   ├─ 同時起動:
   │   ├─ Agent A: トレンドリサーチャー
   │   └─ Agent C: ユーザーインプット分析（インプットがある場合のみ）
   │         │
   │   Agent Aの結果を受けて同時起動:
   │   ├─ Agent B: 深掘りリサーチャー
   │   └─ Agent D: 批判的分析エージェント
        │
   Phase 2: ネタ評価・選定
   └─ 評価エージェントが上位3件を選定
        │
   Phase 3: 投稿文生成
   └─ 3ネタそれぞれについて投稿案を生成
        │
   出力: output/x-posts/YYYY-MM-DD-{slug}.md
```

## Phase 1: ネタ探索

### Agent A: トレンドリサーチャー

- WebSearchで直近24-48時間のテック/AIトレンドを収集
- 検索クエリ例: 「AI 最新ニュース」「エンジニアリング トレンド」「プログラミング 話題」
- 英語圏のトレンドも拾う（Hacker News、Reddit r/programming等）
- 5-10件のネタ候補をリスト化

### Agent B: 深掘りリサーチャー

- Agent Aの結果を受け取り、各ネタの背景・文脈を調査
- 「なぜ今これが話題なのか」「技術的に何が新しいのか」を掘り下げる
- WebFetchで元記事を読み、要点を整理

### Agent C: ユーザーインプット分析（条件付き起動）

- ユーザーがテーマ・記事URL・メモを渡した場合のみ起動
- インプットからX投稿に適したネタの切り口を抽出
- インプットがない場合はスキップし、Agent A/Bの2並列で動く

### Agent D: 批判的分析エージェント

- Agent Aが収集したネタに対して反対意見・批判的視点を調査
- 「本当にそうか？」「誇張されていないか？」「別の解釈はないか？」を検証
- 情報の信頼性（ソースの質、データの裏付け）をチェック
- 各ネタに対して「批判的視点メモ」を付与

### 並列構成

```
同時起動: Agent A + Agent C（独立）
    ↓ Agent Aの結果を受けて
同時起動: Agent B + Agent D（独立）
```

## Phase 2: ネタ評価・選定

評価エージェントがAgent B・Dの結果を統合し、全ネタを採点して上位3件を選定する。

### 評価基準（5段階スコア）

| 基準 | 観点 |
|------|------|
| 会話誘発力 | 問いかけ・議論になりやすいか。リプライ(+75)を狙えるか |
| 独自性 | 他アカウントと差別化できる切り口があるか |
| テーマ適合度 | テック/AI・プロダクティビティの軸に合っているか |
| 批判的深度 | 批判的分析で面白い視点が出たか。「実は...」の切り口があるか |
| 鮮度 | 今投稿する価値があるか（話題のタイミング） |

### 出力

- 上位3件のネタを選定
- 各ネタに「選定理由」「推奨する投稿の切り口」を付与してPhase 3へ渡す

## Phase 3: 投稿文生成

### 生成ルール（Xアルゴリズム最適化）

- **構造**: 結論 → 理由 → 問いかけ（会話誘発のため末尾に問いを置く）
- **トーン優先度**: 問いかけ > 実務共有 > 解説
- **文字数**: 単発は280字以内、深いテーマはスレッド（4-8投稿）も提案
- **ハッシュタグ**: 0-2個（3個以上はペナルティ）
- **外部リンク**: 本文に含めない（リプライに配置する旨を戦略メモに記載）
- **批判的視点の活用**: Agent Dの分析を織り込み、「通説 vs 実態」の対比で議論を促す

### 各案の出力フォーマット

```markdown
## 投稿案 1: {ネタタイトル}

### 投稿文
（そのままコピペ可能な投稿文）

### 戦略メモ
- 狙うシグナル: （リプライ、ブックマーク等）
- 推奨投稿時間:
- 投稿後アクション: （リプライ対応、リンク補足等）
- この形式が有効な理由:
```

## 起動仕様

### トリガー

「X投稿」「ポスト作成」「ツイート」等のキーワード

### 入力パターン

| パターン | 例 | 動作 |
|---------|---|------|
| テーマなし | 「X投稿作って」 | トレンドからネタ探索 |
| テーマ指定 | 「RAGについてX投稿」 | テーマに沿ったネタ探索 |
| インプット付き | 「この記事をもとにX投稿 {URL}」 | Agent C起動 + トレンド探索 |

### ファイル構成

- `.claude/commands/x-post.md` — スキル定義

### 保存先

`output/x-posts/YYYY-MM-DD-{slug}.md`

## アルゴリズム最適化の根拠

リサーチ（2026-03-24）に基づく主要な設計判断:

- **会話誘発を最優先**: 著者のリプライ返信(+75)はいいね(+0.5)の150倍の重み
- **テキスト主体**: Xではテキスト投稿が動画を30%上回る
- **ハッシュタグ制限**: 3個以上で-40%ペナルティ
- **外部リンク排除**: 本文内リンクでリーチ30-50%減少
- **批判的視点の活用**: 「通説 vs 実態」の対比は引用ツイート(いいねの25倍)を誘発しやすい
