# ディープリサーチ

## 入力の取り扱い

ユーザーの発話（例:「○○について調べて」）または、command wrapper 経由で
渡された引数から、調査テーマを抽出する。テーマが曖昧な場合は、調査範囲・
期間・観点を確認してから着手する。

抽出したテーマをもとに、6エージェント体制で多角的な調査を行う。

## フェーズ1: 並列調査（3エージェント同時起動）

3つのサブエージェントを **並列** で起動する:

### リサーチャーA — 最新情報収集
WebSearch と WebFetch を使い、テーマに関する最新の情報を収集する。

**収集対象:**
- 直近1年以内の記事・ニュース・発表
- 公式ドキュメント・リリースノート
- 業界動向・トレンド

**出力フォーマット:**
```
## 最新情報
- [情報1] (出典: URL, 日付)
  信頼度: ★★★★☆
- [情報2] ...
```

### リサーチャーB — 技術的背景
WebSearch を使い、テーマの技術的背景と歴史的文脈を調査する。

**収集対象:**
- 基本概念・仕組みの解説
- 歴史的な経緯・進化の過程
- 関連技術との比較・ポジショニング
- 技術的なメリット・デメリット

**出力フォーマット:**
```
## 技術的背景
### 概要
...
### 関連技術との比較
| 項目 | テーマ | 代替A | 代替B |
|------|--------|-------|-------|
```

### リサーチャーC — 批判的分析
WebSearch を使い、テーマに対する批判的な視点を収集する。

**収集対象:**
- 反対意見・懐疑的な見方
- 既知の課題・制約・限界
- 失敗事例・アンチパターン
- リスク・注意点

**出力フォーマット:**
```
## 批判的分析
### リスク
- [リスク1]: [詳細]
### 課題
- [課題1]: [詳細]
### 反対意見
- [意見1]: [根拠]
```

## フェーズ2: 統合（シンセサイザー）

フェーズ1の3つの出力を受け取り、以下を行う:

1. **主要な発見事項を5-7項目に整理する**
2. **情報の一貫性チェック** — 3エージェント間で矛盾する情報があれば明示する
3. **構造化されたレポート骨子を作成する**

## フェーズ3: レビュー（レビュアー）

統合レポートを以下の観点で検証する:

- **事実の正確性**: 出典が明示されているか、信頼できるか
- **論理的一貫性**: 結論が根拠から導かれているか
- **網羅性**: 重要な視点が欠落していないか

**判定:**
- ✅ 合格 → フェーズ4へ
- 🔄 差し戻し → 具体的な改善点を指示してフェーズ1を再実行（最大1回）

## フェーズ4: レポート作成（レポートライター）

最終レポートを以下のフォーマットで作成する:

```markdown
# リサーチレポート: [テーマ]
日付: YYYY-MM-DD

## エグゼクティブサマリー
[3-5行で結論を述べる]

## 主要な発見事項
1. ...
2. ...

## 詳細分析
### 最新動向
...
### 技術的背景
...
### リスクと課題
...

## 結論と推奨事項
...

## 参考文献
- [タイトル](URL) — 概要
```

**保存先:** `output/research/YYYY-MM-DD-{テーマの英語slug}.md`
