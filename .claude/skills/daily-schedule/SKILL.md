# デイリースケジュール作成

## 手順

### 0. 現在時刻の確認

- 現在時刻を `Asia/Tokyo` タイムゾーンで取得する（`date` コマンド等）
- 取得した時刻を **起動時刻** として記録し、以降のすべてのステップで基準とする
- **スケジュール開始時刻**: 起動時刻を繰り上げた直近の15分刻み（例: 10:07 → 10:15、14:00 → 14:00）

### 1. 情報収集（並列実行）

以下を同時に実行する:

**タスクA: Googleカレンダー確認**
- `gcal_list_events` で本日の **残りの** 予定を取得する
  - calendarId: "primary"
  - timeMin: **現在時刻** (ISO 8601, Asia/Tokyo タイムゾーン)
  - timeMax: 本日 23:59:59 (ISO 8601, Asia/Tokyo タイムゾーン)
  - ※ 進行中の予定（開始済み・未終了）も含まれる
- `gcal_find_my_free_time` で **現在時刻以降の** 空き時間を確認する

**タスクB: コンテキスト読み込み**
- `00_context/memories/preferences.md` を読んでユーザーの好みを確認する
- `00_context/memories/context-log.md` で前日の未完了タスク・保留事項を確認する

### 2. タスクヒアリング

ユーザーに今日取り組みたいタスクを確認する。
既存のカレンダー予定と未完了タスクを提示し、追加タスクがあるか聞く。

### 3. タスク分類

各タスクを以下の3軸で分類する:

| 軸 | 高 | 低 |
|----|----|----|
| 緊急度 | 今日中に必要 | 期限に余裕あり |
| 重要度 | 成果に直結 | なくても回る |
| 認知負荷 | 深い思考が必要 | ルーティン的 |

アイゼンハワーマトリクスで優先順位を決定:
- 🔴 緊急×重要 → 最優先で実行
- 🟡 重要×非緊急 → 計画的に取り組む
- 🟠 緊急×非重要 → 効率化・委任を検討
- ⚪ 非緊急×非重要 → 今日は見送り

### 4. スケジュール生成

15分刻みのタイムブロックでスケジュールを生成する:

**配置ルール:**
- スケジュール開始時刻（手順0で算出）より前の時間帯にはタスクを配置しない
- **起動時刻が12:00より前の場合:**
  - **午前（起動時刻〜12:00）**: 認知負荷の高いタスク（集中作業、創造的作業）
  - **午後（13:00〜）**: ミーティング + ルーティン作業
- **起動時刻が12:00以降の場合:**
  - 午前/午後の区分を適用せず、残り時間で認知負荷の高い順にタスクを配置する
- 既存のカレンダー予定を尊重し、空き時間にタスクを配置する
- タスク間に5分のバッファを設ける
- 90分以上の連続作業には休憩を挟む

**出力フォーマット:**

起動時刻が12:00より前の場合（午前/午後セクション分割）:
```
## 本日のスケジュール（YYYY-MM-DD HH:MM 時点）

### 午前
- HH:MM-HH:MM 🔴 [タスク名] — [概要]
- HH:MM-HH:MM 🟡 [タスク名] — [概要]

### 午後
- 13:00-14:00 📅 [カレンダー予定]
- 14:00-15:30 🟡 [タスク名] — [概要]
- 15:35-16:30 🟠 [タスク名] — [概要]
```

起動時刻が12:00以降の場合（時系列のみ）:
```
## 本日のスケジュール（YYYY-MM-DD HH:MM 時点）

- HH:MM-HH:MM 📅 [カレンダー予定]
- HH:MM-HH:MM 🔴 [タスク名] — [概要]
- HH:MM-HH:MM 🟡 [タスク名] — [概要]
```

### 5. ユーザー承認

生成したスケジュールをユーザーに提示し、調整の要望があれば対応する。

### 6. カレンダー登録

ユーザーの承認後、`gcal_create_event` で各タスクをカレンダーに登録する。

**colorId ルール:**
| 種別 | colorId | 色 |
|------|---------|-----|
| 🔴 緊急×重要 | 11 | 赤 |
| 🟡 重要×非緊急 | 5 | 黄 |
| 🟠 緊急×非重要 | 6 | 橙 |

### 7. 保存

結果を `output/YYYY-MM-DD-schedule.md` に保存する。
