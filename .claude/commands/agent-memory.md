# AI記憶管理

「$ARGUMENTS」の内容を適切なメモリファイルに保存する。

## 処理フロー

### 1. 内容の分類

以下の4カテゴリに自動分類する:

| カテゴリ | 保存先 | 該当例 |
|---------|--------|--------|
| ユーザーの好み・設定 | `00_context/memories/preferences.md` | 「朝型です」「TypeScript派」「敬語不要」 |
| 意思決定 | `00_context/memories/decisions.md` | 「Xの方針をYに決めた」「Aは不採用にした」 |
| セッションコンテキスト | `00_context/memories/context-log.md` | 「明日Aの続きをやる」「Bが途中」 |
| 判断基準 | `00_context/memories/case-judgment-framework.md` | 「こういう場合はXを選ぶ」「Yの判断基準」 |

分類に迷った場合はユーザーに確認する。

### 2. 重複チェック

保存先ファイルを読み、同じ内容が既に記録されていないか確認する。

- **完全一致**: 「既に記録されています」と報告。保存しない。
- **類似内容**: 既存の記録を提示し、更新が必要か確認する。

### 3. 矛盾チェック

既存の記録と矛盾する内容でないか確認する。

- **矛盾あり**: 既存の記録を提示し、どちらを採用するか確認する。
- **更新の場合**: 古い記録を新しい内容で置き換える。

### 4. 保存

適切なファイルに追記する。

**preferences.md の場合:**
```markdown
- [内容] (YYYY-MM-DD 記録)
```

**decisions.md の場合:**
```markdown
| YYYY-MM-DD | [決定事項] | [理由] | [コンテキスト] |
```

**context-log.md の場合:**
```markdown
- [内容] (YYYY-MM-DD 記録)
```

**case-judgment-framework.md の場合:**
```markdown
## [判断カテゴリ]
- **条件**: [こういう場合]
- **判断**: [こうする]
- **理由**: [なぜなら]
(YYYY-MM-DD 記録)
```

### 5. 確認

保存完了をユーザーに報告する:
- 保存先ファイル名
- 保存した内容の要約
- 分類理由（必要に応じて）
