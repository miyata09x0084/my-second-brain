# X投稿テキスト生成

「$ARGUMENTS」を元に、X（Twitter）投稿用のテキストを生成する。

## 入力パターン

以下のいずれかを受け付ける:

1. **テーマ指定**: テーマやトピックのキーワード（例: 「Claude Codeの便利な使い方」）
2. **ファイル指定**: 記事やリサーチ結果のファイルパス（例: `output/articles/2024-01-01-example.md`）
3. **自由テキスト**: そのまま投稿したい内容の下書き

## フェーズ1: 素材収集

### ファイル指定の場合
- 指定されたファイルを読み込み、要点を抽出する

### テーマ指定の場合
- WebSearch でテーマに関する最新情報を1回検索し、要点を把握する

### 自由テキストの場合
- 下書きの意図・主張を分析する

## フェーズ2: 投稿テキスト生成

素材を元に、**3パターン**の投稿テキストを生成する。

**生成ルール:**
- 各パターンは140文字以内（日本語）/ 280文字以内（英語）を目安とする
- ハッシュタグは最大2つまで（多すぎると逆効果）
- URLプレースホルダが必要な場合は `[URL]` と表記する
- 絵文字は控えめに（最大2つ）

**3パターンの切り口:**

| パターン | 切り口 | 例 |
|---------|--------|-----|
| A: 学び共有型 | 知見・発見を共有する | 「〜を試してみたら、〜だった」 |
| B: 問題提起型 | 疑問や課題を投げかける | 「〜って、実は〜じゃないですか？」 |
| C: 実用Tips型 | 具体的なTipsを提示する | 「〜するなら、〜がおすすめ」 |

## フェーズ3: ユーザー選択

3パターンをユーザーに提示する:

```
## X投稿テキスト候補

**A: 学び共有型**
> {テキストA}

**B: 問題提起型**
> {テキストB}

**C: 実用Tips型**
> {テキストC}

---
どれを使いますか？（A/B/C）
修正したい場合はそのまま指示してください。
```

## フェーズ4: 最終調整

ユーザーが選択・修正指示を出したら:

1. 選択されたパターンをベースに最終調整する
2. 文字数を表示する
3. クリップボードにコピーしやすいよう、コードブロックで最終テキストを表示する

```
## 最終テキスト（{文字数}文字）

\`\`\`
{最終投稿テキスト}
\`\`\`

↑ このテキストをコピーしてXに投稿してください。
```
